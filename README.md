# Loan API

REST API для подачи и обработки заявок на займ.
Стек: **Yii2 · PostgreSQL · Nginx · Docker Compose · PHP 8.2**


## Запуск проекта

```bash
# 1. Клонировать репозиторий
git clone <repo-url>

# 2. Собрать образы, установить зависимости, применить миграции
make build
```

`make build` выполняет три шага последовательно:

| Шаг | Команда | Что делает |
|-----|---------|------------|
| 1 | `docker compose up -d --build` | Собирает образы и поднимает контейнеры |
| 2 | `make composer` | Устанавливает PHP-зависимости через Composer |
| 3 | `make migrate` | Применяет миграции БД |

После запуска API доступен на **http://localhost** (порт 80).

## Эндпоинты

### POST /requests — подача заявки на займ

**Тело запроса:**

```json
{
  "user_id": 1,
  "amount": 3000,
  "term": 30
}
```

**Успешный ответ (HTTP 201):**

```json
{
  "result": true,
  "id": 42
}
```

**Ответ при ошибке (HTTP 400):**

```json
{
  "result": false
}
```

Условия отказа:
- Поля отсутствуют, не являются целыми числами или равны нулю/отрицательны.
- У пользователя уже есть одобренная заявка.

---

### GET /processor?delay=5 — обработка заявок

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| delay | int | Задержка в секундах перед принятием решения (эмуляция времени обработки) |

**Ответ (HTTP 200):**

```json
{
  "result": true
}
```

Поведение:
- Все заявки со статусом `pending` обрабатываются последовательно внутри одного вызова.
- Каждая заявка получает статус `approved` (10 % вероятность) или `declined` (90 %).
- У одного пользователя не может быть более одной одобренной заявки.
- Эндпоинт безопасен при одновременных вызовах: `SELECT … FOR UPDATE SKIP LOCKED` гарантирует, что каждую заявку обрабатывает ровно один воркер; уникальный частичный индекс в БД — дополнительная защита от двойного одобрения.

## Примеры запросов

```bash
# Подать заявку
curl -X POST http://localhost/requests \
     -H "Content-Type: application/json" \
     -d '{"user_id": 1, "amount": 3000, "term": 30}'

# Запустить обработку с задержкой 2 секунды
curl "http://localhost/processor?delay=2"
```

## Архитектура

```
app/
├── config/          # Конфигурации (web, console, db)
├── controllers/     # HTTP-слой: RequestsController, ProcessorController
├── models/          # ActiveRecord: LoanRequest
├── repositories/    # Слой доступа к данным + интерфейс
├── services/        # Бизнес-логика + интерфейсы
├── migrations/      # Миграции БД
├── tests/           # Юнит-тесты (PHPUnit)
└── web/             # Точка входа (index.php)
```

Принципы:
- **SOLID** — контроллеры, сервисы, репозитории разделены; зависимости внедряются через интерфейсы (DI-контейнер Yii2).
- **DRY** — повторяющаяся логика вынесена в сервисы и репозиторий.
- **KISS** — минимум абстракций; каждый класс решает одну задачу.
